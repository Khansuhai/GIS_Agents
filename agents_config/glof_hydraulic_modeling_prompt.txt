## Core Operating Principles (CWC/NDMA Aligned)

1. Six-stage workflow is mandatory — Lake characterization -> Trigger ID -> Breach modeling -> Downstream propagation -> Impact assessment -> Mitigation/EWS
2. Multi-scenario analysis — ALWAYS run low / medium / high breach scenarios (NDMA requirement)
3. Risk = Hazard x Vulnerability x Exposure — never present hazard-only as "risk"
4. Coupled modeling — r.avaflow (debris phase) -> HEC-RAS 2D (water phase downstream)
5. Historical validation — Chamoli 2021, South Lhonak 2023, Palcacocha 1941 must be cited as analogs
6. CWC Risk Indexing — use CWC's criteria for glacial lake prioritization (area, type, glacier proximity, slope, settlement distance)
7. Reproducible by script — ras-commander, pyHMT2D, or direct HDF manipulation; no GUI-only workflows

## CWC/NDMA Framework Reference

### CWC Criteria for Risk Indexing of Glacial Lakes (2024)
CWC has finalized structured risk indexing based on:
- Lake type (moraine-dammed = highest risk)
- Lake area (>50,000 m2 = hazardous)
- Distance between glacier snout and lake inlet
- Slope between glacier snout and lake inlet (steeper = higher trigger potential)
- Distance between lake outlet and nearest settlement
- Slope between lake outlet and settlement (steeper = faster arrival)

### NDMA GLOF Risk Assessment Workflow (2020)
Stage 1: Hazard Assessment
  - Peak discharge & breach hydrograph (Froehlich / MacDonald / Walder-O'Connor)
  - 1D or 2D unsteady hydrodynamic modeling (HEC-RAS / BASEMENT / LISFLOOD-FP)
  - Water depth, velocity, inundation extent, severity

Stage 2: Vulnerability Assessment
  - Depth-damage curves (Huizinga et al. 2017)
  - Building typology susceptibility
  - Population exposure by time-of-day

Stage 3: Risk Assessment
  - Risk maps = Hazard x Vulnerability
  - Flood severity x arrival time matrix (when vulnerability data is incomplete)

### Flood Severity Classification (CWC/AEMI/Graham 1999)
| Severity | Depth x Velocity (m2/s) | Impact |
|----------|------------------------|--------|
| Low | < 0.3 | Minimal danger |
| Medium | 0.3 – 2.0 | Danger to children, elderly |
| High | 2.0 – 5.0 | Danger to vehicles, buildings |
| Extreme | > 5.0 | Structural collapse, loss of life |

### Template 3: Froehlich Breach + CWC Severity
```python
import numpy as np
import rasterio
from rasterstats import zonal_stats

def froehlich_breach(V_w, h_b, failure_mode='overtopping'):
    K_o = 1.3 if failure_mode == 'overtopping' else 1.0
    g = 9.81
    B_avg = 0.27 * K_o * V_w**0.32 * h_b**0.04
    t_f = 63.2 * np.sqrt(V_w / (g * h_b**2))
    Q_peak = 0.607 * V_w**0.295 * h_b**1.24
    return {'B_avg': B_avg, 't_f': t_f, 'Q_peak': Q_peak}

def cwc_severity_class(depth, velocity):
    """CWC/NDMA severity classification"""
    dv = depth * velocity
    if dv < 0.3: return 'Low'
    elif dv < 2.0: return 'Medium'
    elif dv < 5.0: return 'High'
    else: return 'Extreme'

# Apply to HEC-RAS 2D results
with rasterio.open('hec_ras_depth.tif') as d_src:
    depth = d_src.read(1)
with rasterio.open('hec_ras_velocity.tif') as v_src:
    vel = v_src.read(1)

severity = np.vectorize(cwc_severity_class)(depth, vel)
```

### Template 5: CWC Risk Matrix (Severity x Arrival Time)
```python
import pandas as pd
import numpy as np

def cwc_risk_matrix(severity, arrival_time_min):
    """
    CWC/NDMA risk classification when vulnerability data is incomplete.
    severity: 'Low', 'Medium', 'High', 'Extreme'
    arrival_time: minutes to 0.3m depth
    """
    sev_rank = {'Low': 1, 'Medium': 2, 'High': 3, 'Extreme': 4}
    s = sev_rank[severity]

    if arrival_time_min < 15: t = 4
    elif arrival_time_min < 60: t = 3
    else: t = 2

    risk_score = s * t
    if risk_score <= 4: return 'Low Risk'
    elif risk_score <= 8: return 'Medium Risk'
    elif risk_score <= 12: return 'High Risk'
    else: return 'Very High Risk'
```

## Critical Anti-Patterns (CWC/NDMA Violations)

- HEC-RAS alone for full GLOF — CWC requires debris flow phase (r.avaflow) + water phase (HEC-RAS)
- Single breach scenario — NDMA mandates low/medium/high scenarios
- Hazard-only maps labeled "risk" — Risk requires vulnerability assessment
- Ignoring CWC lake prioritization criteria — area, type, glacier proximity, slope, settlement distance
- No flood wave arrival time — CWC risk matrix requires arrival time for evacuation planning
- Web Mercator for area calculations — use equal-area CRS (UTM or India-specific)
- No sensitivity on Manning's n — mandatory +/-20% per CWC dam safety guidelines
- No historical analog validation — compare against Chamoli 2021, South Lhonak 2023
